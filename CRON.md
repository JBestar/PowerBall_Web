# 5분 정각 추첨 실행 (Cron 설정)

추첨이 **실제로 5분, 10분, 15분 … 정각(XX:00, XX:05, XX:10, … XX:55)**에 진행되도록 하려면, 아래 둘 중 하나로 **매 5분마다** 실행되도록 설정하세요.

---

## 서버에 작업 스케줄러·관리 권한이 없을 때

Windows **작업 만들기** 권한이 없거나, 서버에 CLI/cron을 둘 수 없을 때는 **방법 2(HTTP)**만 쓰면 됩니다. **서버 쪽에 새 “작업”을 만들 필요가 없습니다.**

1. 서버의 `.env`에 `CRON_DRAW_KEY=` 로 긴 비밀 문자열을 넣습니다.  
2. [cron-job.org](https://cron-job.org), [EasyCron](https://www.easycron.com/) 같은 **외부 무료 cron**에 가입합니다.  
3. 새 작업으로 아래 URL을 **GET**, 주기는 **1분마다** 권장(5분 슬롯이 아닌 분에는 서버가 추첨만 생략합니다).

   `https://실제도메인/cron/draw?key=위에서_넣은_비밀문자열`

4. 사이트가 **HTTPS**로 열려 있어야 하고, 방화벽이 **외부에서 웹 접속**을 막지 않아야 합니다.

**참고**: 본인 **집 PC**의 작업 스케줄러로 `curl`만 돌리는 방법도 있지만, PC를 끄면 추첨이 멈추므로 **외부 cron 서비스**가 더 낫습니다. 서버 관리자에게 부탁할 수 있다면, 관리자 계정으로 **방법 1(작업 스케줄러 또는 `php spark draw:run`)**을 등록해 달라고 요청하면 됩니다.

---

## 방법 1: CLI 명령 (서버에 쉘 접속 가능할 때)

매 5분마다 `draw:run` 명령을 실행합니다.

### 상시 프로세스 (한 번만 실행해 두면 됨)

프로젝트 루트의 **`draw_daemon.php`**는 5분 경계까지 대기한 뒤 추첨을 반복합니다. 서버에서 CMD로 다음만 실행해 두면, 브라우저 접속 없이 추첨이 계속 진행됩니다.

```bash
php draw_daemon.php
```

Windows에서는 `draw_daemon.bat`을 더블클릭해도 됩니다. 테스트로 한 번만 실행하려면 `php draw_daemon.php once` (5분 슬롯일 때만 추첨).

> 이 저장소에 `spark` 실행 파일이 없으면 `php spark draw:run` 대신 위 데몬 또는 아래 crontab의 **`php draw_daemon.php once`**(1분마다)를 쓸 수 있습니다.

### Linux / Mac (crontab)

```bash
crontab -e
```

다음 한 줄 추가 (경로는 실제 프로젝트 루트로 변경):

```
*/5 * * * * cd /path/to/powerball && php spark draw:run
```

### Windows Server / Windows (작업 스케줄러)

`php`가 PATH에 없을 수 있으므로 **`php.exe` 전체 경로**를 쓰는 것이 안전합니다. (XAMPP 예: `C:\xampp\php\php.exe`)

**동작**은 다음처럼 설정합니다.

| 항목 | 값 (예시) |
|------|-----------|
| 프로그램/스크립트 | `C:\xampp\php\php.exe` |
| 인수 추가 | `spark draw:run` |
| 시작 위치(작업 디렉터리) | 프로젝트 루트, 예: `D:\xampp\htdocs\powerball` |

**트리거(중요)**  
`draw:run`은 **분이 0, 5, 10, … 55일 때만** 추첨합니다. 작업 스케줄러에서 “5분마다 반복”을 켜면 **첫 실행 시각**이 :03이면 이후가 :08, :13 …이 되어 **한 번도 추첨이 안 될 수 있습니다.**

- **권장**: 트리거를 **1분마다** 실행하도록 설정하세요. `draw:run`이 5분 슬롯이 아니면 그냥 종료하므로 부하가 거의 없습니다.
- 또는 트리거 시작 시각을 **:00 또는 :05 등 5분 경계**에 맞춘 뒤 “5분마다 반복”을 사용합니다.

**작업 만들기 절차 요약**

1. `Win + R` → `taskschd.msc` → Enter  
2. **작업 만들기**(“기본 작업 만들기”보다 권한·반복 설정이 쉬움)  
3. **일반**: 필요 시 “가장 높은 권한으로 실행”  
4. **트리거**: “매일” 등 원하는 시작 시각 후 → **고급**에서 **반복 작업 간격: 1분**, **기간: 무기한**  
5. **동작**: 위 표대로 `php.exe` / `spark draw:run` / 시작 위치 = 프로젝트 루트  
6. 저장 후 **작업 선택 → 실행**으로 한 번 테스트 → 명령 프롬프트에 출력되거나 DB에 `draw_results`가 쌓이는지 확인  

**방법 2(HTTP)**는 Windows에서도 동일합니다. IIS/Apache만 켜져 있으면 **외부 스케줄러(cron-job.org 등)**가 `https://도메인/cron/draw?key=...`를 1분 또는 5분마다 호출해도 됩니다(5분 슬롯이 아니면 응답만 하고 추첨은 생략).

---

## 방법 2: HTTP 호출 (외부 cron 서비스 사용 시)

cron-job.org, EasyCron 등에서 **매 5분**에 아래 URL을 GET으로 호출합니다.

```
https://도메인/cron/draw?key=비밀키
```

### 비밀키 설정

`.env` 파일에 다음 추가 (값은 임의의 긴 문자열로 설정):

```
CRON_DRAW_KEY=원하는비밀문자열
```

호출 URL의 `key=원하는비밀문자열`이 위 값과 일치해야 동작합니다. 키가 없거나 틀리면 403 Forbidden이 반환됩니다.

---

## 동작 요약

- **5분 단위가 아닌 시각**에 호출되면 추첨은 하지 않고, “Not a 5-min slot” 등으로 응답합니다.
- **정각(XX:00, XX:05, …)**에 호출되면 해당 시각의 추첨이 1회 실행·저장됩니다.
- 이미 해당 시각 추첨이 있으면 새로 만들지 않고 기존 결과를 반환합니다.
- `drawn_at`은 항상 5분 단위(XX:00, XX:05, …)로만 저장됩니다.
